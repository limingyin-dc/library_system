# 图书馆系统启动指南

## 环境准备

确保已激活虚拟环境：

```bash
venv\Scripts\activate
```

## 执行步骤

### 1. 生成数据库迁移文件

```bash
python manage.py makemigrations accounts books borrow catalog purchase readers
```

### 2. 应用迁移到数据库

```bash
python manage.py migrate
```

### 3. 创建管理员账号

```bash
python manage.py createsuperuser
```

按提示输入用户名、邮箱（可留空）、密码。

### 4. 启动开发服务器

```bash
python manage.py runserver
```

## 访问地址

| 页面 | 地址 |
|------|------|
| 系统首页 | http://127.0.0.1:8000 |
| 后台管理 | http://127.0.0.1:8000/admin |

## 本次改动记录

### 问题背景

采购管理存在逻辑缺陷：图书字段强制关联已有图书，但采购时书还没到货不可能已在系统中；采购状态缺少不合格/退货选项；借阅期限和逾期罚金缺乏说明。

### 修改的文件

| 文件 | 改动内容 |
|------|----------|
| `apps/purchase/models.py` | `book` 字段改为可选（`blank=True`）；新增 `rejected`（不合格/退货）状态 |
| `apps/purchase/forms.py` | 调整字段顺序，`book` 字段加帮助文字说明到货后再关联 |
| `templates/purchase/purchase_form.html` | 显示帮助文字和必填标记，底部加操作说明提示 |
| `templates/purchase/purchase_list.html` | 状态用彩色 badge 区分；新增"图书档案"列显示是否已关联 |

### 采购状态说明

| 状态 | 含义 |
|------|------|
| 待处理 | 刚录入，尚未下单 |
| 已订购 | 已向供应商下单，等待到货 |
| 已入库 | 图书到货质检通过，已关联图书档案 |
| 不合格/退货 | 到货后质检不合格，需退货 |
| 已取消 | 采购取消 |

### 借阅期限与逾期说明

借阅期限默认 **30天**，配置在 `library_system/settings.py`：

```python
BORROW_DAYS = 30      # 借阅天数
FINE_PER_DAY = 0.5    # 逾期每天罚金（元）
```

逾期处理逻辑：
- 每次打开借阅管理列表时自动将超期未还记录标记为"已逾期"
- 还书时自动计算罚金（逾期天数 × 每日罚金），显示在还书确认页
- 还书成功后罚金金额记录在借阅记录中

### 应用本次改动

如果数据库已存在，需要执行迁移：

```bash
venv\Scripts\activate
python manage.py makemigrations purchase --name="add_rejected_status_optional_book"
python manage.py migrate
```
